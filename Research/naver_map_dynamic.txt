<동적 map 작동시키는 원리-구식 버전>

동적 map을 연동시키는 구식 방법은 index.html에 다 때려박는것이다
근데 이렇게 했더니 성공은 했지만 position이 absolute일때만 위치가 먹히고
대신 지도 확대축소는 안되는 현상이 있었다
그래서 이렇게도 가능했다 정도만 알아두고 실제로는 아래에 있는
리액트를 적극적으로 사용하는 신식 버전을 사용하도록 한다

어떤 원리로 해야하냐면
1. index.html의 header에
<script type="text/javascript" src="https://openapi.map.naver.com/openapi/v3/maps.js?ncpClientId=실제 자신의 아이디"></script>
   이걸 집어넣는다 이때 정적지도 할떄는 X-NCP-APIGW-API-KEY-ID=gzqf4on71t
   이렇게 썼으니까 낯설거임 하지만 ncpClientId 옆에도 내가 받은 id쓰는거 맞음
   그리고 이때는 이상하게도 비번 없이 아이디만 넣어도됨
   그래서 결국 이렇게된다
    <script type="text/javascript" src="https://openapi.map.naver.com/openapi/v3/maps.js?ncpClientId=gzqf4on71t"></script>

2. 그다음에 map이랑 mapOptions 변수 설정하는 부분인데
   이걸 리액트에다 하는게 아님 api 설명서 보면 body 안에다가 이걸 하고있어서
   리액트에 해야한다고 착각할수 있겠는데 이 변수 설정까지도
   index.html 안에 써야한다 index.html의 <body>태그 안에 쓰는것이다
   그래서 결국 이렇게된다
   <body>
    <script>
     var mapOptions = {
        center: new naver.maps.LatLng(37.3595704, 127.105399),
        zoom: 10
     };

     var map = new naver.maps.Map('map', mapOptions);
     </script>

3. 실제로 인쇄하는 부분 역시 body안에다 쓴다
   리액트에다 썼던건 없애야됨
   <body>
        <div id="map" style="width:30%;height:400px;margin-left:40%;margin-top:25%;position:absolute;border:2px solid white;border-radius:5px"></div>
    <script> ~ </script>

<동적 map 작동시키는 원리-신식 버전>

신식 버전은 index.html header안에 지도불러오는 script 하나만 놓고
나머지는 다 jsx에서 해결하는 것이다 그래서 1번은 구식과 같은 원리임 복붙

어떤 원리로 해야하냐면
1. index.html의 header에
<script type="text/javascript" src="https://openapi.map.naver.com/openapi/v3/maps.js?ncpClientId=실제 자신의 아이디"></script>
   이걸 집어넣는다 이때 정적지도 할떄는 X-NCP-APIGW-API-KEY-ID=gzqf4on71t
   이렇게 썼으니까 낯설거임 하지만 ncpClientId 옆에도 내가 받은 id쓰는거 맞음
   그리고 이때는 이상하게도 비번 없이 아이디만 넣어도됨
   그래서 결국 이렇게된다
    <script type="text/javascript" src="https://openapi.map.naver.com/openapi/v3/maps.js?ncpClientId=gzqf4on71t"></script>

2. 설정하는 부분은 jsx에서 constructor과 componentDidMout를 이용해서 한다

3. 실제로 지도 출력하는건 그냥 render의 return 안에다가
<div id="map"></div> 이렇게 하면 된다

=>그렇게 성공한 총 코드는 다음과 같다
import React, { PureComponent } from 'react';
import "./MapProblem.css"

class MapProblem extends PureComponent {

    constructor(){
        super();
        this.state={
            "mapOptions":[{
                "center": "",
                "zoom": "",
            }],
            "map": ""
        }
    }

    componentDidMount(){
       this.setState({mapOptions:  [{center:new window.naver.maps.LatLng(37.3595704, 127.105399), zoom : 10 }], ...this.state});
       this.setState({map:new window.naver.maps.Map('map', this.mapOptions), ...this.state});
       //그리고 여기서 안됐던 이유는 잘못된 예제를 가져왔기 때문이었음 weather측에서 안쓰는 trnasaction배열 내에도 temp있지만 실제로 쓴건 바깥의 temp임
       //그래서 이거는 실제로는 어떻게 해야하냐면 mapOptions배열을 통째로 갈아껴야됨 배열내의 center변수만 바꾸는건 불가능하다는 소리다
       //그리고 저 두번째인자로 ...this.state를 써야했음 꼭 필요한지는 아직 실험해보지 않았다   
 }
    render() {
        return (
            <div className="MapProblem">
                <div className="MapText">
                <h2 id="mapTitle">공격할 지역을 선택하세요  ex)</h2>
                <div id="mapExample">
                    <img id="mapImg" src='https://naveropenapi.apigw.ntruss.com/map-static/v2/raster?w=300&h=300&center=127.1054221,37.3591614&level=16&X-NCP-APIGW-API-KEY-ID=gzqf4on71t&X-NCP-APIGW-API-KEY=ddnnOWuUjtES0k0JO21ecv9ojJw58tGt0sCAWUNI' alt="지도"/>
                    </div>
                </div>
                <div id="map"></div>
            </div>
        );
    }
}

export default MapProblem;
